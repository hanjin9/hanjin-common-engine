## 1. GLWA 플랫폼 전체 구조

### 1.1 부모-자식 앱 아키텍처

```
GLWA (부모 앱 / 메인 플랫폼)
│
├── 건강 카테고리 1: 숨 (호흡) ← 현재 MVP 개발 중
├── 건강 카테고리 2: 수면
├── 건강 카테고리 3: 명상
├── 건강 카테고리 4: 헬스 / 운동
├── 건강 카테고리 5: 심리 / 정신 건강
├── 건강 카테고리 6~10: (추후 확장)
└── 통합 VIP 멤버십 (10단계)
```

**핵심 원칙:** GLWA는 10개 건강 카테고리를 통합 관리하는 플랫폼이며, 각 자식 앱은 독립 운영 후 GLWA와 연동/통합 가능한 구조로 설계한다.

### 1.2 숨호흡 MVP의 포지셔닝

- GLWA 10개 건강 카테고리 중 **첫 번째 카테고리 (호흡)**
- 빠른 수익화를 위해 독립 앱으로 먼저 출시
- 멤버십 구조는 GLWA와 **별도 독립** (구독 레벨 4~6단계)
- 나중에 GLWA와 연동 또는 통합 가능 (parentProjectSlug 필드로 연결 예약됨)

> **주의:** 연회비/포인트 임계값은 현재 가상 수치이며, 정식 오픈 전 관리자 대시보드에서 수정 가능. Black Platinum(11단계)은 추후 활성화 예정으로 DB에 예약만 해둠.

### 2.2 단계 확장 원칙

- 단계 수는 **DB row 추가/비활성화**만으로 변경 가능 (코드 수정 불필요)
- 최대 단계 수 제한 없음 (동적 구조)
- 각 단계별 **이미지 업로드 슬롯** 존재 (관리자가 럭셔리 이미지 삽입 가능)
- 단계명, 색상, 연회비, 혜택, 포인트 임계값 모두 관리자 대시보드에서 실시간 편집

## 3. 멤버십 성격 및 운영 철학

### 3.1 멤버십 유형

GLWA 멤버십은 **건강 수련 클럽 멤버십**이다. 단순 구독 서비스가 아닌, 지속적인 건강 수련 활동을 기반으로 한 **커뮤니티 회원권** 성격을 가진다.

**벤치마킹 레퍼런스:**
- 로타리클럽 / 라이온스클럽 — 등급 구조, 커뮤니티 활동 기반 승급
- 럭셔리 리조트 멤버십 (Aman, Six Senses 등) — 단계별 차별화된 혜택, 프라이빗 서비스
- 호텔 VIP 프로그램 — 전담 매니저, 우선 예약권, 리조트 크레딧

**우리만의 차별점:** 건강 수련 기간과 활동 점수가 승급의 핵심 기준. 돈만 내면 올라가는 구조가 아닌, **실제 건강 수련 참여도**가 반영됨.

### 3.2 승급 기준 (다중 트리거)

| 승급 기준 | 설명 |
|-----------|------|
| 연회비 납부 | 단계별 연회비 납부 시 등급 유지/승급 조건 충족 |
| 건강 수련 기간 | 1년/2년/3년 이상 지속 수련 시 포인트 자동 부여 |
| 활동 점수 (포인트) | 미션 완수, 건강 체크, 커뮤니티 활동 등 누적 |
| 관리자 수동 승급 | 특별 기여자, VIP 초대, 특수 상황 시 관리자 직접 지정 |

> **중요:** 사용자가 임의로 등급을 변경할 수 없음. 관리자만 수동 변경 가능하며, 알고리즘 자동 승급 또는 관리자 수동 지정 두 가지 경로만 존재.

## 4. 숨호흡 MVP 구독 레벨 구조

### 4.1 단계 정의

| 순서 | 단계 키 | 표시명 | 성격 | 상태 |
|------|---------|--------|------|------|
| 1 | free | Free | 무료 기본 체험 | 운영 |
| 2 | bronze | Bronze | 소액 유료 구독 | 운영 |
| 3 | silver | Silver | 중간 유료 구독 | 운영 |
| 4 | gold | Gold | 고급 유료 구독 | 운영 |
| 5 | premium | Premium | 최고급 구독 | 운영 |
| (6) | vip | VIP | GLWA 연동 후 활성화 예정 | **예약 비활성** |

### 4.2 숨호흡 앱 멤버십 원칙

- GLWA VIP 멤버십과 **완전 독립** 운영
- 구독 서비스 성격 (멤버십 아님) — 월 구독료 기반
- 나중에 GLWA 통합 시 VIP 레벨 활성화
- 단계 수는 4~6단계로 유연하게 조정 가능

## 5. 기술 구현 현황

### 5.1 DB 테이블 구조

| 테이블명 | 역할 |
|---------|------|
| `project_membership_tiers` | 프로젝트별 단계 정의 (동적, 무제한 확장) |
| `project_user_memberships` | 사용자별 현재 단계 기록 |
| `membership_policy_history` | 정책 변경 이력 (변경 전/후 값, 변경 사유, 시각) |

### 5.2 관리자 기능 (구현 완료)

- `adminGetTierStats` — 프로젝트별 단계 분포 통계 조회
- `adminUpdateTier` — 단계 정책 편집 (혜택/연회비/색상/포인트) + 자동 이력 기록
- `adminUpdateUserTier` — 사용자 단계 수동 변경
- `adminGetPolicyHistory` — 정책 변경 이력 조회
- `adminSeedTiers` — 초기 시드 데이터 삽입 (GLWA 10단계 + 숨호흡 5단계)

### 5.3 GitHub 저장 현황

- **hanjin-common-engine** — 관리자 엔진 (현재 작업 중)
- **glwa-wellness-app** — GLWA 앱 원본 (10단계 vipTierSystem.ts 업데이트 완료)
