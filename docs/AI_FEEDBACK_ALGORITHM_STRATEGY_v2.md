## 1. 핵심 방향성 (대표님 지시 요약)

> "AI가 분석해서 자동으로 알고리즘으로 피드백을 보내야 한다.  
> 잘 하는 사람에게는 이벤트·상·포인트, 못 하는 사람에게는 격려 피드백.  
> 1차·2차·3차 피드백 단계로 나눠서 오늘/주간/월간 단위로 관리."

## 2. 3단계 피드백 시스템 (기존 + 신규 전략)

### 2.1 기존 3단계 구조 (GLWA 기존 코드 기반)

| 단계 | 명칭 | 트리거 | 내용 |
|------|------|--------|------|
| **1단계** | 즉시 자동 피드백 | 측정 직후 0.1초 | 음성/텍스트 즉시 응답 |
| **2단계** | 프리미엄 심화 피드백 | 일 1회 자동 | 생체데이터 종합 분석 + 생활 처방 |
| **3단계** | VIP 전문가 피드백 | 주 1회 / 이상 감지 시 | 1:1 전문가 코칭 연결 |

### 2.2 신규 추가 전략 (대표님 지시)

| 단계 | 명칭 | 주기 | 내용 |
|------|------|------|------|
| **오늘 피드백** | 일일 성과 리포트 | 매일 오후 8시 자동 | 오늘 수련 점수 + 어제 대비 변화 + 격려/칭찬 메시지 |
| **주간 피드백** | 주간 랭킹 리포트 | 매주 일요일 | 주간 TOP 10 발표 + 하위 10명 격려 + 포인트 지급 |
| **월간 피드백** | 월간 종합 분석 | 매월 1일 | 월간 성장 그래프 + 등급 승급 여부 + 이벤트 보상 |

## 3. 순위 시스템 (랭킹 알고리즘)

### 3.1 점수 산출 공식

```
종합 점수 = (수련 일수 × 10) + (평균 웰니스 점수 × 2) + (연속 수련 보너스 × 5) + (미션 완료 보너스 × 3)
```

### 3.2 등급별 자동 보상 알고리즘

| 순위 | 등급 | 자동 액션 |
|------|------|-----------|
| TOP 1~3 | 챔피언 | 🏆 배지 + 포인트 500P + 관리자 알림 + 축하 AI 메시지 |
| TOP 4~10 | 우수 수련자 | 🥇 배지 + 포인트 200P + 칭찬 AI 메시지 |
| TOP 11~30 | 성장 수련자 | ⭐ 배지 + 포인트 100P + 격려 AI 메시지 |
| 하위 20% | 격려 대상 | 💪 특별 격려 메시지 + 맞춤 수련 가이드 + 관리자 알림 |
| 30일 미접속 | 이탈 위험 | 🔔 재참여 유도 메시지 + 특별 이벤트 초대 |

### 3.3 자동 격려 메시지 알고리즘 (LLM 기반)

```typescript
// 하위 사용자 자동 격려 로직 (예시)
if (userRank > totalUsers * 0.8) {
  await invokeLLM({
    messages: [
      { role: "system", content: "당신은 따뜻한 건강 코치입니다. 수련을 포기하려는 사람에게 진심 어린 격려 메시지를 보내세요." },
      { role: "user", content: `${userName}님의 최근 7일 수련 점수: ${recentScore}. 개인화된 격려 메시지를 50자 이내로 작성하세요.` }
    ]
  });
}
```

## 4. 측정 데이터 분류 체계 (관리자 대시보드 표시 기준)

### 4.1 생체 데이터 카테고리

| 카테고리 | 측정 항목 | 우선순위 |
|----------|-----------|----------|
| **호흡** | 호흡수, 호흡 깊이, 호흡 패턴, 산소포화도 | ★★★★★ |
| **심장** | 심박수, 심박 변이도(HRV), 안정시 심박 | ★★★★★ |
| **수면** | 수면 시간, 수면 질, 깊은수면 비율, 기상 시간 | ★★★★☆ |
| **운동** | 운동 시간, 운동 강도, 걸음수, 칼로리 | ★★★★☆ |
| **스트레스** | 스트레스 지수, 명상 시간, 이완도 | ★★★☆☆ |
| **영양** | 식사 시간, 수분 섭취, 영양 균형 | ★★★☆☆ |

### 4.2 관리자 대시보드 AI 분석 섹션 개선 방향

현재 (단촐한 4개 카드) → 개선 목표:

```
[AI 분석 현황 섹션]
├─ 오늘 피드백 발송 현황 (발송수 / 열람수 / 반응률)
├─ 주간 랭킹 TOP 5 미리보기 (이름 + 점수 + 변화)
├─ 주의 필요 사용자 (하위 20% + 30일 미접속)
├─ 1차/2차/3차 피드백 단계별 발송 통계
├─ 측정 데이터 카테고리별 평균 점수 레이더 차트
└─ 이번 주 자동 보상 지급 현황 (포인트 + 배지)
```

## 7. 기술 스택 (구현 시 사용)

- **스케줄러:** Heartbeat (periodic-updates.md 참조) — 매일 오후 8시, 매주 일요일 자동 실행
- **LLM:** `invokeLLM()` (server/_core/llm.ts) — 개인화 메시지 생성
- **알림:** `notifyOwner()` + 이메일 — 랭킹 발표, 이상 감지 시 관리자 알림
- **DB:** `feedbackLogs` + `biodataRecords` + 신규 `userRankings` 테이블
- **포인트 시스템:** 신규 `userPoints` 테이블 (향후 마이그레이션)
